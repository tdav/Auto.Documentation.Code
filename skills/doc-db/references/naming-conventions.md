# Соглашения именования таблиц и колонок

> **Настраиваемый файл.** Значения ниже — **дефолты-гипотезы** (выведены из
> анализа типового ADO.NET-кода). Перед серьёзным прогоном `doc-db` сверьте их
> со своей базой и поправьте. Особенно подтвердите **языковую раскладку**
> `NAME00..NAME05` — она угадана и может не совпадать с вашей.

## Префиксы таблиц

| Префикс | Значение | Схема (по умолчанию) |
|---|---|---|
| `SV_*` | Справочники (Static Values) | `DICT` |
| `TV_*` | Транзакционные таблицы (Transaction Values) | `APP` |

_Поправьте под свои префиксы и схемы._

## Эвристики колонок

| Шаблон имени | Описание |
|---|---|
| `SV_ID`, `TV_ID`, `<PFX>_ID` | **PK** — суррогатный идентификатор (`NUMBER(19)`). |
| `*_GUID` | Глобальный уникальный идентификатор записи (`RAW(16)`/GUID); используется для синхронизации между средами. |
| `*_PREVIOUSID` | Самоссылка на предыдущую версию записи (паттерн «история через linked list»). |
| `*_RECORDSTATUS` | Soft-delete: `1` — активна, `0` — удалена/архивная, `2` — изменен |
| `*_USER` | ID пользователя последней операции (audit). |
| `*_DATEENTER`, `*_DATEUPDATE` | Дата создания / последнего изменения (audit). |
| `*_COMMENT` | Произвольный комментарий. |
| `*_NAME00`..`*_NAME05` | Мультиязычные наименования (см. раскладку ниже). |
| `*_SYSTEM`, `*_SYSTEMID` | FK на справочник «Систем». |
| `*_REQUESTID` + `*_REQUESTGUID` | Двойная FK на таблицу заявок: numeric + GUID. |
| `*_PERSONID` | FK на справочник физлиц. |

## Языковая раскладка `NAME00..NAME05` (ПОДТВЕРДИТЕ!)

> ⚠️ Угадано эвристически. Замените на реальную раскладку вашего проекта.

| Колонка | Язык/локаль (гипотеза) |
|---|---|
| `*_NAME00` | Локаль 0 |
| `*_NAME01` | Локаль 1 |
| `*_NAME02` | Локаль 2 |
| `*_NAME03` | Локаль 3 |
| `*_NAME04` | Локаль 4 |
| `*_NAME05` | Локаль 5 |

> Раскладка `NAME00..05` — наименования на разных языках/локалях проекта;
> настраивается под вашу базу.

## Эвристика FK (два сигнала)

1. Имя колонки совпадает с PK другой таблицы (`*_PERSONID` → `SV_PERSON.SV_ID`).
2. Колонка встречается в JOIN-условиях SELECT.

Все выведенные FK помечаются словом **«предположительно»** — не убирать без
подтверждения от DBA.
