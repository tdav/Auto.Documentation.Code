# XML-doc теги Microsoft — справочник для doc-cs

Канонический набор тегов из
[learn.microsoft.com/ru-ru/dotnet/csharp/language-reference/xmldoc/recommended-tags](https://learn.microsoft.com/ru-ru/dotnet/csharp/language-reference/xmldoc/recommended-tags).
Загружается по требованию, когда нужно вспомнить точное назначение тега.

## Таблица тегов

| Тег | Назначение | Применяется к |
|---|---|---|
| `<summary>` | Краткое описание (для IntelliSense) | Любой публичный член |
| `<remarks>` | Развёрнутые сведения; Markdown в `CDATA` для DocFX | Тип, метод, свойство |
| `<param name="x">` | Описание параметра; имя обязано совпадать с сигнатурой (иначе CS1572/CS1573) | Метод/конструктор |
| `<paramref name="x"/>` | Ссылка на параметр внутри текста | Внутри `<summary>`/`<remarks>` |
| `<returns>` | Описание возвращаемого значения | Метод |
| `<exception cref="T">` | Какие исключения и в каких случаях | Метод/свойство/индексатор/событие |
| `<value>` | Что представляет значение свойства | Свойство |
| `<typeparam name="T">` | Описание параметра-типа дженерика | Generic |
| `<typeparamref name="T"/>` | Ссылка на параметр-тип | Внутри текста |
| `<example>` + `<code>` | Пример использования + кодовый блок | Метод/тип |
| `<c>` | Инлайн-код | Внутри текста |
| `<see cref="…"/>`, `<see href="…">`, `<see langword="…"/>` | Ссылка на код / URL / ключевое слово | Внутри текста |
| `<seealso cref="…"/>` | Ссылка в разделе «См. также» | Любой |
| `<para>` / `<br/>` | Абзац / перенос | Внутри текста |
| `<list type="bullet\|number\|table">` | Списки/таблицы (DocFX v2 поддерживает ровно 2 колонки в `table` — issue dotnet/docfx#5492) | Внутри текста |
| `<inheritdoc cref="…" path="…"/>` | Наследование документации | Override-ы, реализации интерфейсов |
| `<include file="…" path="…"/>` | Подключение комментариев из внешнего XML | Любой |

## Правила Microsoft (мягкие требования)

- Документировать все публичные типы и их публичные члены; приватные — опционально.
- Минимум — `<summary>`; полные предложения, точка в конце.
- При мульти-параметрах — несколько `<param>` (иначе предупреждение компилятора CS1573).
- HTML-теги `<b>`, `<i>`, `<u>`, `<br/>`, `<a>` валидны.
- Экранирование: `<` → `&lt;`, `>` → `&gt;`.
- `cref` валидируется компилятором (предупреждение CS1574 на битую ссылку) — ссылайтесь через `cref`, а не через голый текст.

## Нюанс `<inheritdoc/>`

Visual Studio автонаследует доку в IntelliSense и Quick Info, но компилятор
**не** кладёт это в выходной XML-файл документации. Цитата Microsoft Learn:
*«это автоматическое наследование применяется только в Visual Studio IDE и не
влияет на XML-файл документации, созданный компилятором»*. Для NuGet-публикации
тег `<inheritdoc/>` нужно ставить явно.

## Примеры «хорошо / плохо»

### `<summary>`
- ❌ `GetUserById` → «Получает пользователя по идентификатору» (тавтология, имя
  уже это сказало).
- ✅ «Загружает запись `DICT.SV_ITEM` по первичному ключу `SV_ID`; используется
  при отрисовке дерева доступов в клиентском приложении оператора».

### `<param>`
- ❌ `<param name="id">Идентификатор.</param>`
- ✅ `<param name="id">Первичный ключ записи `DICT.SV_ITEM` (`SV_ID`,
  `NUMBER(19)`).</param>`

### `<exception>`
- ❌ `<exception cref="OracleException">Ошибка БД.</exception>`
- ✅ `<exception cref="OracleException">Возникает при нарушении уникальности
  `SV_GUID` или недоступности схемы `DICT`.</exception>`

### `<returns>`
- ❌ «Возвращает результат.»
- ✅ «Количество вставленных строк (ожидается 1); при нарушении уникальности
  метод выбрасывает исключение, а не возвращает 0.»

### `<list>` в XML-doc
- ❌ `<list type="table">` с тремя колонками — DocFX v2 отрендерит только две.
- ✅ `<list type="bullet">` для перечислений; таблицы выноси во внешний `.md`.
